# Phase 3: Polish — P2 편의 기능

## 목표
MVP(Phase 1) + 핵심 기능(Phase 2)이 안정적으로 동작하는 상태에서,
사용 편의성과 운영 효율을 높이는 부가 기능을 추가한다.

## 전제 조건
Phase 2가 완료되어 모든 P0/P1 기능이 동작하는 상태.

## 예상 소요: 1~2주 (필요한 것만 선택적으로 구현)

## 중요: Phase 3는 전부 구현할 필요 없다
아래 작업들은 우선순위가 낮으므로, 필요성이 느껴지는 것만 골라서 구현한다.
연구실 피드백을 받아 우선순위를 재조정할 수 있다.

---

## 작업 목록

### 3-1. 세그멘테이션 결과 다운로드
**참조 스펙**: product-specs/segmentation-overlay.md §4

- 전체 레이블맵 NIfTI 다운로드
- 개별 클래스 바이너리 마스크 다운로드
- `GET /api/segments/{result_id}/download` 엔드포인트
- Frontend에 다운로드 버튼 추가

### 3-2. Inference 히스토리
**참조 스펙**: product-specs/inference-pipeline.md §3

- 영상별 이전 inference 결과 목록
- 과거 결과 다시 불러오기
- `GET /api/inference/history?image_id=...` 엔드포인트
- Frontend에 히스토리 패널 또는 드롭다운

### 3-3. 모델 메모/태그
**참조 스펙**: product-specs/model-management.md §1

- 각 모델에 메모 및 태그 추가
- model_notes/ 디렉토리에 JSON으로 관리
- 모델 선택 UI에서 태그 뱃지 표시

### 3-4. 성능 최적화
- 대용량 볼륨 progressive loading
- gzip 압축 조건부 적용
- 인메모리 inference (predict_single_npy_array) 검토
- Frontend 렌더링 성능 프로파일링

### 3-5. GPU 상태 모니터링 UI
- 하단 상태바에 GPU 사용량 표시
- 캐시된 모델 목록 표시
- 현재 inference 진행상황

### 3-6. 사용자 경험 개선
- 색상 피커 (클래스별 색상 변경)
- 클래스별 개별 투명도 조절
- 선택적 지우기 (특정 클래스만 지우기)
- 3D 편집 (여러 슬라이스 동시 편집)
- 키보드 단축키 도움말 패널

### 3-7. 운영 편의
- 업로드 파일 자동 정리 (7일 이상 된 파일 삭제)
- 모델 warm-up (앱 시작 시 자주 쓰는 모델 미리 로드)
- 로그 수준 설정 (환경변수)
- 에러 리포팅 (선택)

## Phase 3 작업 선택 기준

"이거 먼저 해야 할까?" 판단 기준:
1. **연구원 3명 이상이 같은 불편함을 호소**하면 → 즉시 구현
2. **데이터 손실 위험**이 있으면 → 즉시 구현 (예: 자동 저장)
3. **10분 이내에 구현 가능**하면 → 바로 구현
4. 나머지는 → 백로그에 기록하고 여유 있을 때 구현

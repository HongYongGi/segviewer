# SECURITY.md — 보안 가이드

## 보안 범위
이 앱은 **연구실 내부 네트워크 전용**이다.
외부 인터넷 접근이 불가능한 환경을 전제로 설계되었다.

따라서 일반적인 웹 보안(HTTPS, CSRF, XSS 등)은 최소 수준만 적용하고,
**데이터 보호**에 집중한다.

---

## 데이터 보호 규칙

### 1. 환자 데이터를 외부로 전송하지 않는다
- 외부 API 호출 금지 (analytics, CDN, 외부 라이브러리 런타임 로딩 등)
- 모든 의존성은 빌드 시 번들링 (런타임 외부 요청 없음)
- Docker 컨테이너의 네트워크를 내부망으로 제한

### 2. NIfTI 파일은 서버 내부에서만 처리한다
- 업로드된 파일은 서버 로컬 디스크에만 저장
- 클라우드 스토리지(S3 등)에 업로드하지 않음
- 결과 파일도 동일

### 3. nnUNet weight는 읽기 전용이다
- weight 디렉토리는 Docker에서 `:ro` (read-only)로 마운트
- 웹앱 코드가 weight를 수정/삭제할 수 없음

## API 보안 (최소 수준)

### 인증 없음 (의도적)
내부망 전용이므로 로그인/인증을 구현하지 않는다.
외부 노출이 필요해질 경우 역방향 프록시(nginx)에서 Basic Auth 또는
OAuth2를 추가한다.

### 입력 검증
- 파일 업로드: 확장자 + nibabel 파싱으로 이중 검증
- 파일 크기: MAX_UPLOAD_SIZE_MB 제한 (기본 500MB)
- Path traversal 방지: image_id에 UUID만 허용 (파일명 직접 사용 금지)
- 세그멘테이션 저장 시: shape 일치, 값 범위 검증

### CORS 설정
```python
# 내부망이므로 모든 origin 허용
allow_origins=["*"]
```
외부 노출 시 반드시 특정 origin으로 제한해야 한다.

## 주의사항

### 이 앱을 외부 인터넷에 노출하지 말 것
- 인증 없음 → 누구나 파일 업로드/다운로드 가능
- HTTPS 미적용 → 데이터가 평문 전송
- 파일시스템 직접 접근 → 악의적 파일 업로드 위험

외부 접근이 필요한 경우 반드시:
1. 역방향 프록시 + HTTPS 적용
2. 인증 시스템 추가
3. 업로드 파일 검증 강화
4. 별도 보안 review 수행

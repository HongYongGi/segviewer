# DESIGN.md — UI/UX 디자인 가이드

## 디자인 원칙

### 1. 의료영상 뷰어의 관례를 따른다
연구자들은 ITK-SNAP, 3D Slicer, OHIF 등 기존 의료영상 뷰어에 익숙하다.
이 앱의 UI는 가능한 한 이런 도구들의 관례를 따라야 혼란을 줄일 수 있다.

- **다크 테마**: 의료영상은 어두운 배경에서 볼 때 눈의 피로가 적다
- **2x2 그리드**: Axial/Coronal/Sagittal/3D 배치는 거의 모든 뷰어의 표준
- **좌클릭 = W/L 조절**: ITK-SNAP, RadiAnt 등의 관례
- **우클릭 = 팬**: 대부분의 뷰어와 동일
- **휠 = 슬라이스 이동**: 보편적 관례

### 2. 정보 밀도를 높인다
연구자는 한 화면에서 많은 정보를 동시에 보기를 원한다.
불필요한 여백, 큰 버튼, 과도한 애니메이션을 피하고
가능한 한 많은 뷰포트 영역을 확보한다.

### 3. 자주 쓰는 기능은 1클릭 이내
- 윈도우/레벨 프리셋 변경: 1클릭
- 모델 선택 + inference 실행: 2클릭 (선택 + Run)
- 브러시/지우개 전환: 1키 (B/E)

### 크기 규칙
| 영역 | 크기 | 비고 |
|------|------|------|
| 상단 툴바 | 높이 48px | 고정 |
| 뷰포트 영역 | 나머지 전체 | 가변, 2x2 균등 분할 |
| 우측 사이드바 | 너비 300px | 접기 가능 |
| 하단 상태바 | 높이 24px | 고정, GPU 상태 표시 |

### 반응형 대응
- **≥ 1280px**: 전체 레이아웃 (2x2 + 사이드바)
- **< 1280px**: 단일 뷰포트 + 탭 전환 (Axial|Coronal|Sagittal|3D)
- **모바일**: 지원하지 않음 (의료영상 뷰어는 데스크탑 전용)

## 컴포넌트 디자인

### 버튼
| 종류 | 용도 | 스타일 |
|------|------|--------|
| Primary | Run Inference, Save | 보라색 배경, 흰 텍스트 |
| Secondary | Upload, Refresh | 테두리만, 회색 텍스트 |
| Danger | Delete (P2) | 빨간색 배경, 흰 텍스트 |
| Ghost | 도구 선택, 뷰 전환 | 배경 없음, 호버 시 약간 밝아짐 |

### 드롭다운
- 모델 선택: 계층적 드롭다운 (Dataset → Configuration → Fold)
- W/L 프리셋: 단순 드롭다운
- 선택 시 즉시 적용 (확인 버튼 불필요)

### 슬라이더
- 투명도, 브러시 크기: 수평 슬라이더
- 현재 값을 슬라이더 옆에 숫자로 표시
- 드래그 중 실시간 반영

### 토스트 메시지
- 성공: 초록색, 3초 후 자동 사라짐 ("저장 완료")
- 에러: 빨간색, 수동 닫기 필요 ("GPU 메모리 부족")
- 정보: 파란색, 5초 후 자동 사라짐 ("TTA 없이 실행됨")
- 위치: 우측 하단

## 뷰포트 내부 디자인

### 뷰포트 오버레이 정보
각 2D 뷰포트에 표시되는 텍스트 오버레이:

```
┌──────────────────────────────────┐
│ Axial [64/128]         W:400 L:40│  ← 좌상단: 방향+슬라이스, 우상단: W/L
│          A                       │
│                                  │
│ R ─── + ─── L        ← 방향 라벨│
│                                  │
│          P                       │
│                         ├──┤     │  ← 우하단: 스케일바 (10mm 등)
│ zoom: 1.5x              10mm    │  ← 좌하단: 줌 레벨
└──────────────────────────────────┘
```

- 텍스트 색상: 흰색, 약간 반투명 (rgba(255,255,255,0.8))
- 폰트: 모노스페이스, 12px
- 크로스헤어: 노란색 점선, 1px

### 세그멘테이션 오버레이
- 각 클래스의 채움: 반투명 (alpha = 전체 투명도 슬라이더 값)
- 경계선: 1px 실선, 해당 클래스 색상 100% 불투명
  (경계선이 있어야 세그멘테이션 범위를 정확히 파악 가능)

## 접근성 (최소 수준)

- 키보드만으로 모든 기능 접근 가능 (Tab 순서 합리적)
- 색약 고려: 컬러맵에 빨강+초록만 의존하지 않도록 형태/패턴 차이 포함 (P2)
- 충분한 대비: 텍스트와 배경의 대비율 4.5:1 이상

# PRODUCT_SENSE.md — 제품 판단 기준

## 이 문서의 목적
"이 기능을 추가해야 할까?", "이 UI가 맞는 걸까?" 같은
제품 판단이 필요할 때 참고하는 기준이다.

## 우리 사용자를 기억하라

### 페르소나: 연구실 연구원
- **배경**: 의료영상 분석 석/박사과정 또는 연구원
- **일상**: 모델 학습 → 결과 확인 → 논문 작성
- **기술 스택**: Python, PyTorch, nnUNet에 능숙. 웹/JS는 비전문
- **목표**: "이 세그멘테이션 결과가 맞는지 빨리 확인하고 싶다"
- **좌절 포인트**: 터미널 명령어 입력 → 파일 경로 복붙 → ITK-SNAP 실행 → 반복

### 이 사용자가 기뻐하는 것
- 업로드 → 결과 확인까지 클릭 3번
- 익숙한 뷰어 레이아웃 (ITK-SNAP 스타일)
- 브러시로 몇 곳 수정 → 바로 NIfTI 저장

### 이 사용자가 짜증나는 것
- 복잡한 설정이 필요한 초기 세팅
- 에러 메시지만 뜨고 해결 방법이 없는 것
- 느린 반응 (클릭 후 3초 이상 무반응)
- 예상과 다른 UI (간이 왼쪽에 있어야 하는데 오른쪽에 있는 것)

## 기능 추가 판단 기준

새 기능을 추가할지 결정할 때 이 질문을 던져라:

### 1. "연구원 5명 중 3명 이상이 쓸 기능인가?"
- YES → 구현 고려
- NO → 보류 (개인 선호는 환경설정으로 해결)

### 2. "이 기능 없이도 앱을 쓸 수 있는가?"
- YES → P2 (나중에)
- NO → P0 또는 P1

### 3. "기존 의료영상 뷰어(ITK-SNAP, 3D Slicer)에 있는 기능인가?"
- YES → 구현 방식도 가능한 한 관례를 따름
- NO → 정말 필요한지 재검토

### 4. "구현 시간 대비 사용 빈도가 높은가?"
- 1시간 구현 / 매일 사용 → 즉시 구현
- 1주 구현 / 가끔 사용 → 보류

## UI 판단 기준

### "어디에 넣을까?" 판단
- 매번 쓰는 기능 → 항상 보이는 곳 (툴바, 사이드바)
- 가끔 쓰는 기능 → 드롭다운, 설정 패널
- 거의 안 쓰는 기능 → 구현하지 않음

### "어떻게 보여줄까?" 판단
- 상태 정보 (현재 W/L, 슬라이스 번호) → 뷰포트 오버레이 텍스트
- 선택 항목 (모델, 프리셋) → 드롭다운
- 켜기/끄기 (클래스 표시) → 체크박스
- 연속값 (투명도, 브러시 크기) → 슬라이더
- 실행 (inference, 저장) → 버튼
